Programmation en C

GCC​

  1. Télécharger GCC

    Se rendre sur Winlibs (https://winlibs.com/) et installer la dernière version de GCC - Windows 64 bits - Sans LLVM.

  2. Installer GCC

    Lancer l’exécutable téléchargé et suivre la procédure d’installation.

  3. Ajouter GCC au path Windows

    • Chercher “Modifier les variables d’environnements du système”
    • Cliquer “Variables d’environnement …”
    • Dans “Variables système”
      • Sélectionner “Path” puis cliquer “Modifier”
        • Cliquer “Nouveau”
        • Saisir le chemin vers gcc.exe (C:<chemin>\mingw64\bin)
        • Cliquer “Ok”
    • Fermer toutes les fenêtres d’ajouts de variables d’environnement
    • Fermer tous les invites de commandes
    • Ouvrir un invite de commande et vérifier la présence de gcc avec gcc --version
      • Si la version de GCC s’affiche, c’est bon.

Commandes de bases de GCC

  • Compiler
C:\Users\John> gcc fichier.c 
# Compile fichier.c et crée a.exe
  • Compiler et choisir le nom du .exe en sortie (-o)
C:\Users\John> gcc fichier.c -o fichier.exe 
# Compile fichier.c et crée fichier.exe

Une fois le fichier.c compilé en fichier.exe, on peut l’exécuter depuis l’invite de commande :

C:\Users\John> fichier.exe

Convention de nommage​

Il existe des conventions de nommage en C, elles définissent la nomenclature à adopter pour les noms de variables et de fonctions.

Les deux conventions majeures sont :

  1. lowerCamelCase

    Exemple :

    • unEntier
    • uneVariableQuiEstUneChaineDeCaractere
    • maFonction
    • fonctionSommeDeTroisEntiers

  2. snake_case

    Exemple :

    • un_entier
    • une_variable_qui_est_une_chaine_de_caractere
    • ma_fonction
    • fonction_somme_de_trois_entiers

Qu'on choisisse lowerCamelCase ou snake_case, peu importe, ce qui compte c'est d'utiliser la même convetion tout au long du programme.

Déclaration et affectation de variables​

  • Une variable déclarée dans une fonction, n'existe que dans cette fonction.
  • Elle ne peut pas être réutilisé dans une autre fonction ou par le programme principal (main).
  • Pour récupérer la valeur d’une variable existant dans une fonction, il faut utiliser le mot clé return. Attention, une fonction le return effectué, le programme est sorti de la fonction.
  • On peut aussi utiliser des variables globales. Ces variables sont accessibles à toutes les fonctions et au programme principal. Cependant, les variables globales, rendent le code difficile à maintenir.
  • Ces mots clés ne peuvent pas être utilisés comme nom de variables ou de fonctions, ils sont réservés : auto - break - case - char - const - default - do - double - else - enum - extern - float - for - goto - if - int - long - register - return - short - signed - sizeof - static - struct - switch - typedef - union - unsigned - void - volatile - while
  • Les pointeurs sont la solution. Les pointeurs permettent à une fonction de modifier la valeur d’une variable, puis on peut la récupérer depuis une autre fonction ou le programme principal sans avoir besoin d’utiliser un return. Le pointeur nous permet de modifier la valeur de la variable, on ne travaille pas sur une copie de la variable. Les pointeurs doivent être maniée avec précaution.

Déclaration de variable :

TYPE nomDeVariable;

Affectation de valeur à une variable :

nomDeVariable = VALEUR;

Exemple :

// Déclaration de ma variable "monEntier" qui est de type entier.
int monEntier;
printf("monEntier vaut : %d \n", monEntier);
// La valeur affichée ici est imprévisible, parfois 0, parfois aléatoire !
// C'est pour cela qu'il faut affecter une valeur à une variable.

// Affectation de ma variable "monEntier" qui vaut 13.
monEntier = 13;
printf("monEntier vaut : %d \n", monEntier);

// Déclaration et affectation de ma variable "monAutreEntier" qui est de type entier et qui vaut 12.
int monAutreEntier = 12;
printf("monAutreEntier vaut : %d \n", monAutreEntier);

Signed vs unsigned

  • Signé : peut contenir des valeurs positives et négatives
  • Non-signé : ne contient que des valeurs positives, mais avec une plage plus large

En C, les nombres sont signés par défaut. Cela signifie que les types numériques comme int, short, long, et char sont interprétés comme signés à moins qu'ils soient explicitement déclarés comme non signés avec le mot-clé unsigned.

Exemple :

  • int x; → Signé (peut contenir des valeurs positives et négatives).
  • unsigned int x; → Non signé (ne contient que des valeurs positives, mais avec une plage plus large).

Exception :

Pour le type char, la signedness par défaut dépend de l'implémentation du compilateur. Certains compilateurs le considèrent comme signé, tandis que d'autres comme non signé. Pour éviter l'ambiguïté, il est préférable d'utiliser explicitement signed char ou unsigned char.

ℹ️

Dans un premier temps, on peut ignorer la notion de signed ou unsigned.

Les formats spécificateurs / jokers / drapeau

  • Entier : int : %d
  • Nombre décimal : float : %f
  • Caractère : char : %c
  • Chaine de caractère : char var[] : %s
  • Adresse mémoire (pointeur) : int *ptr : %p

On se sert des formats spécificateurs lors d’un affichage via printf().

Compléments

  • Les variables doivent toutes être déclarées au début de la fonctionne ou du main. Pas en cours de programme.

  • Il existe la subtilité des types signed vs unsigned mais on ne s'en occupera pas ici par soucis de simplicité.

  • Il existe les types short, long, double, long double qui reprennent les types présentés plus haut, avec plus ou moins d'espace mémoire, et plus ou moins d'octets affectés par variable selon le type.

  • On peut utiliser des variables dites "globales" mais c'est à déconseiller.

    • Une telle variable est déclaré au début du programme, en dehors de toute fonction. Mais est accessible par toutes les fonctions.
    • C'est à éviter pour des raisons de performances et de complexité dans le code.

  • L'utilisation d'une variable globale static force le compilateur à rendre cette variable accessible uniquement aux fonctions du fichier dans laquelle elle est définie.

    • Il est possible que par défaut toutes les variables soit considérées comme statiques par le compilateur.
    • Une variable globale (déclarée sans static) est accessible à tous les fichiers .c du programme, à condition d'utiliser le mot-clé extern dans les autres fichiers pour y accéder.
    • Une variable globale static est accessible uniquement dans le fichier .c où elle est définie. Elle n'est pas visible dans les autres fichiers, même avec extern.
    • Exemple :
    // Fichier1.c
    #include <stdio.h>
    #include "fichier1.h"

    int counter = 0;  // Visible par tous les fichiers .c 
    // static int counter = 0; // Visible uniquement par fichier1.c 

    void increment() {
        counter++;
    }

    // Fichier1.h
    #ifndef FICHIER1_H
    #define FICHIER1_H

    void increment();  // Déclaration de la fonction

    #endif

    // main.c
    #include <stdio.h>
    #include "fichier1.h"

    extern int counter; // On récupère la variable externe

    int main()
    {
        printf("%d \n",counter);
        increment(); // On accède à la fonction de fichier1.c
        printf("%d",counter);

        return 0;
    }

    Résultat :

    0
    1

  • En revanche, l'utilisation du mot clé static lors de la déclaration d'une variable locale dans une fonction, permet de ne pas détruire cette variable lors de la fin de l'exécution de la fonction. Cela permet donc de conserver la valeur d'une variable local.

    • Sans le mot clé static.

      #include <stdio.h>

      void fonction(){
          int nb_fois_appel = 0;
          nb_fois_appel++;
          printf("%d \n", nb_fois_appel);
      }

      int main()
      {
          fonction();
          fonction();
          fonction();
          return 0;
      }

      // Résultat
      1
      1
      1

    • Avec le mot clé static.

      #include <stdio.h>

      void fonction(){
          static int nb_fois_appel = 0;
          nb_fois_appel++;
          printf("%d \n", nb_fois_appel);
      }

      int main()
      {
          fonction();
          fonction();
          fonction();
          return 0;
      }

      // Résultat
      1
      2
      3

  • Le mot clé const permet de considérer une variable comme immuable. Une fois initialiser, on ne pourra pas la modifier.

    const int x = 10;
    x = 20; // Erreur : x est constant.

  • Exemple de déclarations et affectations de variables. Plus affichage.

    void lesDifferentsTypesDeVariables()
    {
        int unEntier = 12 ;
        float unNombreDecimal = 15.1234 ;
        char unCaractere = 'A' ;
        char unAutreCaractere = 65 ;
        char uneChaineDeCaracteres[] = "ABCDEF" ;
        const float PI = 3.14 ;

        printf("unEntier vaut : %d \n", unEntier);
        printf("unNombreDecimal vaut : %f \n", unNombreDecimal);
        printf("unNombreDecimal arrondi a deux chiffres apres la virgule vaut : %.2f \n", unNombreDecimal);
        printf("unNombreDecimal converti en entier vaut : %d \n", (int)unNombreDecimal);
        printf("unCaractere vaut : %c \n", unCaractere);
        printf("unAutreCaractere vaut : %c \n", unAutreCaractere);
        printf("uneChaineDeCaractereVaut : %s \n", uneChaineDeCaracteres);
        printf("La variable constante PI est immuable et vaut : %f \n", PI);
    }

Lecture de données clavier​

La saisie clavier se fait à l'aide du mot clé scanf, d'un format spécificateur %X, et de l'adresse de la variable (pas de sa valeur !).

Pour faire référence à l’adresse d’une variable et pas sa valeur, on utilise le caractère &.

scanf("%d",&maVariable); // Récupération d'un entier et affectation de cette valeur à la variable maVariable. 
void lectureDeDonneesClavier(){
    int maVariable = 0;
    printf("Saisir un nombre entier : ");

    scanf("%d",&maVariable);

    printf("maVariable vaut : %d \n", maVariable);
}
  • maVariable : fait référence à valeur de la variable (dans un premier temps : 0).
  • &maVariable : fait référence à l'adresse où est stockée la variable.

Avec la saisie clavier, on décide donc d'affecter à l'adresse de la variable maVariable, la valeur saisie par l'utilisateur, un entier, via le format spécificateur %d.

Les opérateurs​

Voici les opérateurs disponibles en C :

  • + : addition

  • - : soustraction

  • * : multiplication

  • / : division (résultat entier) : 5 / 2 = 2

  • % : modulo (reste d’une division) : 5 % 2 = 1

  • = : affectation

  • += X : incrémentation de X

  • -= X : décrémentation de X

  • ++ : incrémentation de 1

  • -- : décrémentation de 1

void lesOperateurs(){

    int maVariable = 0; // maVariable vaut 0

    printf("maVariable vaut : %d \n", maVariable);
    printf("maVariable + 5 vaut : %d \n", maVariable + 5); // maVariable vaut 0

    maVariable = maVariable + 10; // maVariable vaut 10
    printf("J'ajoute 10 et maVariable vaut maintenant : %d \n", maVariable); 

    maVariable += 7 ; // maVariable vaut 17
    printf("J'ajoute 7 et maVariable vaut desormais : %d \n", maVariable);

    maVariable ++; // maVariable vaut 18
    printf("J'incremente de 1 et maVariable vaut finalement : %d \n", maVariable);

    maVariable --; // maVariable vaut 17
    printf("Je decremente de 1 et maVariable vaut finalement : %d \n", maVariable);
}

// ----- Début Affichage console ----- //
maVariable vaut : 0 
maVariable + 5 vaut : 5 
J'ajoute 10 et maVariable vaut maintenant : 10 
J'ajoute 7 et maVariable vaut desormais : 17 
J'incremente de 1 et maVariable vaut finalement : 18 
Je decremente de 1 et maVariable vaut finalement : 17
// ----- Fin Affichage console ----- //

Le IF - ELSE IF - ELSE

⚠️

Si l'instruction dans le if ne fait qu'une ligne, on peut omettre les accolades mais c'est fortement déconseillé.

void lesConditionsIfElseIfElse(){
    int maVariable = 20;

    if(maVariable == 0)
    {
        printf("maVarible vaut 0 \n");
    }
    else if(1 <= maVariable && maVariable <= 10)
    {
        printf("maVarible est comprise entre 1 et 10 \n");
    }
    else if(10 < maVariable && maVariable < 20)
    {
        printf("maVarible est comprise entre 11 et 19 \n");
    }else
    {
            printf("maVariable vaut au moins 20 \n");
    }
    
    int monNombre = 15;
    int boolean = -1; // Un "booléen" qu'on initialise à -1. 0:false 1:true

    // On teste si monNombre vaut 15. Si c'est vrai on retourne 1, si c'est faux on retourne 0. boolean prend la valeur retournee.
    // On utilise un opérateur ternaire pour réaliser cette opération. Non recommandé !
    boolean = (monNombre == 15) ? 1 : 0; 
    
    // Equivalent à 
    if(monNombre == 15){
            boolean = 1;
    }else{
            boolean = 0;
    }

    // On teste si boolean est vrai (==1). Si c'est vrai on affiche un certain message, si c'est faux on en affiche un autre.
    // Opérateur ternaire. Non recommandé.
    (boolean == 1) ? printf("monNombre vaut bien 15 \n") : printf("monNombre ne vaut pas 15 \n");
    
    // if. Recommandé.
    if (boolean == 1){
                printf("monNombre vaut bien 15 \n");
        }else{
                printf("monNombre ne vaut pas 15 \n");
        }
}

Le SWITCH - CASE

  • Particulièrement utile pour les menus.
  • Dans les case, on ne peut tester que des valeurs entières.
  • break permet de sortir du switch une fois la condition réalisée.
  • default regroupe l’ensemble des autres cas possibles. Equivalent du else.
void lesConditionsSwitchCase(){
    int autreVariable = 5 ;

    switch(autreVariable)
    {
        case 0:
            printf("autreVariable vaut 0 \n");
            break;
        case 1:
            printf("autreVariable vaut 1 \n");
            break;
        case 2:
            printf("autreVariable vaut 2 \n");
            break;
        default :
            printf("autreVariable ne vaut pas 0, 1 ou 2 \n");
            break;
    }
}

// ----- Début Affichage console ----- //
autreVariable ne vaut pas 0, 1 ou 2
// ----- Fin Affichage console ----- //

Les boucles​

Les boucles permettent de répéter une instruction un certain nombre de fois.

Il existe deux types de boucles :

  • while
  • do - while
  • for

Boucle WHILE

La boucle while vérifie la condition puis s'exécute si la condition est vérifiée.

  • Syntaxe générique

    while(condition){
            instruction;
    }

  • Exemple :

    void lesBouclesWhile(){
        int i = 0;
        while( i < 5 )
        {
            printf("%d. Bonjour tout le monde :) \n",i);
            i++;
        }
        printf("\n");
    }

    // ----- Début Affichage console ----- //
    0. Bonjour tout le monde :)
    1. Bonjour tout le monde :)
    2. Bonjour tout le monde :)
    3. Bonjour tout le monde :)
    4. Bonjour tout le monde :)
    // ----- Fin Affichage console ----- //

Boucle DO-WHILE

La boucle do-while s'exécute au moins une fois avant de vérifier la condition puis de boucler si celle-ci est vérifiée.

  • Syntaxe générique :

    do{
            instruction;
    while(condition);

  • Exemple :

    void lesBouclesDoWhile(){
            /*do{
                instruction;
          }while(condition);
        */
        int cpt = 10;
        
        do{
            printf("%d. Bonsoir tout le monde :) \n",cpt);
            cpt++;
        }while( cpt < 5);
        
        printf("\n");
    }

    // ----- Début Affichage console ----- //
    10. Bousoir tout le monde :)
    // ----- Fin Affichage console ----- //

Boucle FOR

La boucle for s'exécute un nombre de fois prédéterminé à l'avance. On précise un début, une fin (exclue) et le pas.

  • Syntaxe générique :

    for(debut ; fin ; pas){
            instruction;
    }

  • Exemple :

    void lesBouclesFor(){
        for(int j=0 ; j < 5 ; j = j+2)
        {
            printf("%d. Salut tout le monde :) \n", j);
        }
    }

    // ----- Début Affichage console ----- //
    0. Salut tout le monde :)
    2. Salut tout le monde :)
    4. Salut tout le monde :)
    // ----- Fin Affichage console ----- //

Les fonctions​

  • Une fonction permet de factoriser du code.

  • Exemples de fonctions :

    • printf()
    • scanf()

  • Formalisme d’une fonction :

    <type_de_retour> <nom_de_la_fonction>(<parametres_entrees>)
    {
            instruction;
            //return variable_de_type_type_de_retour
    }

  • Si une fonction ne retourne rien, on peut indiquer void dans l'emplacement <type_de_retour>.

  • Une variable déclarée dans une fonction n'existe que dans cette fonction. On appelle cela des variables locales et elles sont détruites lors que la fonction à fini de s'exécuter.

  • L'utilisation du mot clé static devant la déclaration d'une fonction force cette fonction à n'être accessible que depuis le fichier dans laquelle elle est définie.

  • L'utilisation du mot clé static devant la déclaration d'une variable dans une fonction permet à cette variable de ne pas être détruite à la fin de l’exécution de la fonction.

  • Les fonctions ne peuvent pas être imbriquées en C.

  • Le prototype d’une fonction (aussi appelée signature) est l’entête d’une fonction :

    • Syntaxe : <type_de_retour> <nom_de_la_fonction>(<parametres_entrees>);
    • Exemple : float aire_cercle(int rayon);

  • Si une méthode est définie après le main, celle-ci ne sera pas utilisable dans le main. Il faut alors, soit :

    • Déclarer la fonction avant le main.

    • Déclarer le prototype de la fonction avant le main, et laisser la fonction après le main.

      void maFonction(); // Prototype

      int main() {
          maFonction(); // Appel valide
      }

      void maFonction() {
          // Définition
      }

  • Exemple :

    float aire_cercle(int rayon){
        const float PI = 3.14;
        float aire = PI * pow(rayon,2); // Besoin de la bibliotèque "math" pour utiliser la fonction "pow" ("#include <math.h>" en haut du programme)
        return aire;
    }

    int bonjour()
    {
        printf("Bonjour ! \n");
        return 1;
    }

    int retourFonctionBonjour = bonjour(); // On récupère la valeur retourne mais on affiche aussi le message "Bonjour !").
    printf("La fonction bonjour() retourne : %d \n", retourFonctionBonjour);

    int rayon = 5;
    printf("Un cercle d'une aire de %d vaut : %.2f \n", rayon, aire_cercle(rayon));

    // ----- Début Affichage console ----- //
    Bonjour ! 
    La fonction bonjour() retourne : 1 
    Un cercle d'une aire de 5 vaut : 78.50 
    // ----- Fin Affichage console ----- //

Programmation modulaire​

  • La modularité c'est utiliser des bibliothèques externes. C’est augmenter les possibilités.
  • Les fichiers en .h ce sont les fichiers de descriptions (prototypes des fonctions).
  • Les fichiers en .c contiennent le contenu du code.

Exemple

  • Fichier d'en-tête : math_utils.h

    Ce fichier contient les prototypes des fonctions.

    #ifndef MATH_UTILS_H
    #define MATH_UTILS_H

    // Prototype d'une fonction pour additionner deux nombres
    int addition(int a, int b);

    // Prototype d'une fonction pour soustraire deux nombres
    int soustraction(int a, int b);

    #endif

  • Fichier d'implémentation : math_utils.c

    Ce fichier contient les définitions des fonctions déclarées dans le .h.

    #include "math_utils.h"

    // Définition de l'addition
    int addition(int a, int b) {
        return a + b;
    }

    // Définition de la soustraction
    int soustraction(int a, int b) {
        return a - b;
    }

  • Programme principal : main.c

    Ce fichier utilise le module défini dans les deux fichiers précédents.

    #include <stdio.h>
    #include "math_utils.h"

    int main() {
        int x = 10, y = 5;

        printf("Addition : %d + %d = %d\n", x, y, addition(x, y));
        printf("Soustraction : %d - %d = %d\n", x, y, soustraction(x, y));

        return 0;
    }

  • Compilation et exécution

    Compilez les fichiers ensemble pour créer un exécutable :

    gcc main.c math_utils.c -o main // Création de l'exécutable
    ./main                          // Exécution de l"exécutable

  • La syntaxe d’un fichier .h est spécifique, on y retrouve des gardes d'inclusion (include guards)

    • Leur objectif est d’éviter les inclusions multiples d'un fichier d'en-tête dans un même fichier source.
    • #ifndef MATH_UTILS_H
      • Signifie : "Si MATH_UTILS_H n'est pas défini..."
      • Cette ligne vérifie si une macro appelée MATH_UTILS_H existe déjà. Si ce n'est pas le cas, cela indique que le fichier d'en-tête n'a pas encore été inclus.
    • #define MATH_UTILS_H
      • Définit la macro MATH_UTILS_H pour indiquer que ce fichier est en cours d'inclusion.
      • Une fois ce fichier d'en-tête inclus, la macro MATH_UTILS_H est définie.
    • Code principal (contenu entre les gardes)
    • #endif
      • Terminer le bloc conditionnel ouvert par #ifndef.
      • Cela marque la fin des gardes d'inclusion.

  • A noter que tout fonctionne sans les #include sur CodeBlocks, c'est une facilité permise par cet IDE, ce ne serait pas forcément le cas sans IDE, avec un éditeur de texte + un compilateur.

  • Pour comprendre la programmation modulaire, on peut tester dans ces fichiers le fonctionnement de variables locales et globales statiques ainsi que de fonctions statiques.

Préprocesseur​

  • Le préprocesseur est un sous programme qui s'exécute avant la compilation.

  • Il opère des changements en suivant des directives qu'on lui indique.

  • A travers les mots clés include ou define, on peut indiquer une certaine configuration.

    • Ces mots clés sont appelés des directives de préprocesseur.

  • #include

    • Ce mot clé permet de faire référence à des bibliothèques, d'inclure un fichier.
      • Si cette bibliothèque est préexistante, on l’appelle en utilisant des < >.
        • Ex : #include <stdio.h>
      • Si cette bibliothèque est externe (développée), on l’appelle en utilisant des " ".
        • Ex : #include "ma_biblio.h"

  • #define

    • Ce mot clé permet d'indiquer au préprocesseur certains besoins de remplacements. Ces remplacements ont lieu avant la compilation.
      • On peut créer des constantes.
        • Ex : #define __LONGUEUR__ 15 : Permet de modifier toutes les occurrences de __LONGUEUR__ par 15.
      • On peut remplacer des fonctions.
        • Ex : #define afficher printf : Permet de remplacer toutes les occurrences de afficher par printf.
      • On peut remplacer des instructions.
        • Ex : #define Coucou printf("Coucou tout le monde\n") : Permet de remplacer toutes les occurrences de Coucou par printf("Coucou tout le monde \n").
      • On peut faire des calculs directement dans les #define. Attention cependant aux types des différentes constantes utilisés. (voir exemple avec les taxes).
    • Il existe des constantes prédéfinis du langage C :
      • __FILE__ : nom du fichier
      • __LINE__ : ligne du fichier
      • __DATE__ : date de compilation
      • __TIME__ : heure de compilation
    • Ce mot clé permet aussi d'éviter les boucles d'inclusion, au sein des fichiers .h, lors d'une programmation modulaire.

      • Lors d'une inclusion, on défini une variable. Ainsi, si on tente à nouveau cette inclusion, il suffit de tester si la variable défini existe déjà, si oui, on ne refait pas l'inclusion.

        #ifndef __BIBLIOTEQUE_H__
        #define __BIBLIOTEQUE_H__

        <instructions>

        #endif  __BIBLIOTEQUE_H__
        • #ifndef NOM et #endif NOM : Ce sont aussi des directives, elles permettent de tester l'existence ou non de la constante NOM. C'est à coupler avec #define NOM.

  • Exemple :

    #define TVA 20

        Coucou;

Pointeurs​

  • Il existe deux notations équivalentes :
    • pointeurP1->nom et (*pointeurP1).nom. La première est plus claire et simple.
  • Pointeur : variable contenant l'adresse d'une autre variable.
  • Un pointeur permet à une fonction de modifier des données.
  • %p : permet d'afficher l'adresse d'une variable ou d'un pointeur.

Variables

  • maVariable : valeur de la variable
  • &maVariable : adresse de la variable

Pointeurs

  • monPointeur : adresse de la variable pointée

  • *monPointeur : valeur de la variable pointée

  • &monPointeur : adresse du pointeur

  • <typeVariableA> *monPointeur = variableA; (comme type pour le pointeur, on indique le type de la valeur vers laquelle le pointeur pointe).

    • *monPointeur = NULL ou *monPointeur = &maVariable

  • Intérêt des pointeurs dans les fonctions :

    • Lorsqu'une fonction fini de s'exécuter, ses variables locales sont supprimées.

    • Il faudrait utiliser un return pour renvoyer une valeur pouvant être récupéré dans le main. Cependant, si on a deux valeurs à retourner, on ne peut pas faire de :

      void fonction(){
              int valeur1 = 1;
              int valeur2 = 2;
              return valeur1;
              return valeur2;
      }
      // Ici, seul le premier return sera exécuté. La fonction renvoie 1.

    • La solution réside dans les pointeurs. On ne passe pas en paramètre la valeur d’une variable mais son adresse. On peut alors modifier sa valeur depuis une fonction, via son adresse.

#include <stdio.h>

/*
    Cette fonction ne permet pas de changer la valeur du nombre passé en paramètre car
    lorsqu'une fonction fini de s'exécuter, les variables locales la composant sont supprimées.
*/
void changementNombreNonFonctionnel(int nombre)
{
    nombre = 10;
}

void changementNombreAvecPointeurs(int *nombre)
{
    *nombre = 20;
}

int main()
{

    int nombreA = 15 ;

    printf("nombreA vaut : %d \n", nombreA); // 15
    printf("L'adresse de nombreA est : %p \n", &nombreA); // imprévisible_1

    changementNombreNonFonctionnel(nombreA);
    printf("nombreA vaut toujours : %d \n", nombreA); // 15

    changementNombreAvecPointeurs(&nombreA);
    printf("Grace aux pointeurs, nombreA vaut maintenant : %d \n", nombreA); // 20

    // Le fait de mettre " * " lors de la déclaration du pointeur a pour but de justement indiquer qu'on a affaire à un pointeur.
    // Dans les faits, c'est plutôt "pointeurVersNombreA" que nous déclarons.
    int *pointeurVersNombreA = NULL;
    pointeurVersNombreA = &nombreA;

    printf("\n\nValeur du pointeurVersNombreA : %p \n", pointeurVersNombreA); // 20
    printf("Adresse de nombreA : %p \n", &nombreA); // imprévisible_1
    printf("Ces deux valeurs sont normalement egales ! En effet, pointeurVersNombreA pointe vers l'adresse de nombreA. \n\n");

    printf("Adresse contenue dans le pointeur : %p \n", pointeurVersNombreA); // imprévisible_1
    printf("Valeur vers laquelle pointe l'adresse indique dans le pointeur : %d \n", *pointeurVersNombreA); // 20

Tableaux​

  • Un tableau est un pointeur (vers son premier élément).

Déclaration tableau

 <type> <nom_tableau>[X] // X représente la taille du tableau. Valeurs aléatoires si pas d'affectation.
int tableauCinqEntier[5];

Dans la mémoire, on aura 5 cases d'affilées.

Initialisation

int tableau[5] = {5, -4, 0, -10, 8};
int tableau[5] = {};  // Toutes les cases valent 0
int tableau[5] = {0}; // Toutes les cases valent 0
int tableau[5] = {4}; // Seul la première valeur vaut 4, le reste vaut 0
int tableau[3,2] = { {1,2}, {3,4}, {5,6} };

Accès au tableau

tableau[X]    // élément d'indice X (X+1ème élement du tableau)
tableau       // adresse du tableau (pas besoin de &, c'est déjà une adresse).
*tableau      // premier élement du tableau
*(tableau+X)  // élément d'indice X

// Notation
tableau[X] <=> *(tableau + X)

Exemple 1 :

#include <stdio.h>

int main()
{

    int tableau[5] = {5, -4, 0, -10, 8};
    printf("valeur: %p \n",tableau);
    printf("valeur : %p \n",&tableau);
    printf("valeur : %p \n",&tableau[0]);
    printf("valeur : %d \n",tableau[0]);
    printf("valeur : %d \n",*tableau);
    printf("valeur : %d \n",tableau[3]);
    printf("valeur : %d \n",*(tableau+1));
    printf("valeur : %d \n",*tableau+1);
    
    return 0;
}

Chaines de caractères​

  • Une chaine de caractères c’est du texte.

  • Une chaine de caractères c'est un tableau de char. Chaque éléments du tableau est un caractère + caractère \0 pour symboliser la fin de la chaine.

  • On ne peut pas changer la taille d'un tableau (d'une chaine de caractères) en cours de programme. On ne peut pas faire :

    char mot[] = "boite";
    mot = "chaussure";

Quelques fonctions de chaines de caractères, issues du module <string.h>.

  • strcpy() : copie d'une chaine dans une autre.

    strcpy(<chaine_destination>,<chaine_source>)

  • strlen() : retourne la longueur d'une chaine. Caractère de fin de chaine \0 non inclus.

  • strcmpt() : comparer deux chaines (ordre alphabétique).

  • strcat() : concaténer deux chaines.

  • strstr() : chercher une chaine dans une autre.

  • strchr() : cherche la 1ere occurrence d'un caractère.

  • sprintf() : écrire dans une chaine. (fonction de stdio.h) -> La différence avec strcpy c'est qu'on peut formater la string.

Structures et types en C​

Structure, type, énumération et union.

  • Structure : Une structure en C est une collection de variables de différents types regroupées sous un même nom, permettant de créer un enregistrement de données.

  • Type : En C, le terme "type" se réfère à la catégorie de données qu'une variable peut stocker, telle que int, float, char, etc.

  • Enumération : Une énumération en C est une liste de valeurs symboliques nommées, représentant un ensemble discret d'options ou de constantes.

  • Union : permet de stocker différents types de données au même emplacement mémoire.

  • On positionne une structure au dessus du main. (voir même dans les fichiers d'entêtes.

  • Une structure permet un regroupement de données.

  • Structure à deux champs

    struct PersonneStructure
    {
        char prenom[50];
        int age;
    };

  • Type (structure que l’on transforme en un type)

    typedef struct PersonneType
    {
        char prenom[50];
        int age;
    }Personne;

    //OU pour simplifier. typedef permet de signifier "Personne" est un alias de "PersonneType". Mais si on a pas l'intention d'utiliser "struct PersonneType" mais simplement l'alias, alors la version suivante est plus simple : 

    typedef struct
    {
        char prenom[50];
        int age;
    }Personne;

  • Enumération

    typedef enum Booleen{
        false, // 0 par défaut
        true   // 1 par défaut
    }Booleen;

    //OU plus simplement : 

    typedef enum{
        false, // 0 par défaut
        true   // 1 par défaut
    }Booleen;

    typedef enum MarqueVoiture{
        RENAULT = 10,
        PEUGEOT = 20,
        CITROEN = 30
    }Marque;

  • Union

    union Nombre {
        int entier;
        float decimal;
    };

    //Les deux membres partagent la même zone mémoire, donc l'union ne peut stocker qu'une seule valeur à la fois. A la différence d'une structure qui a un emplacement mémoire par champ.

Allocation dynamique de mémoire​

  • Il faut importer la bibliothèque <stdlib.h>
  • sizeof(<donnee ou type_de_donnee>) : retourne la taille en octets d'une donnée ou d'un type.
  • malloc(<taille_en_octets>) : allouer dynamiquement une zone mémoire.
    • <taille_en_octets> = <donnee> * sizeof(<type>)
  • free(<donnee_allouee>) : libère la mémoire alouée dynamiquement.
  • calloc(<donnee>, sizeof(<type>)) : alloue et initialise tout à 0.
  • realloc(<donne>, <taille_en_octets>) : réalloue un espace mémoire (on peut l'agrandir
    • <taille_en_octets> = <donnee> * sizeof(<type>)

    int *liste_age_personne = NULL;

    nombrePersonnes = 2;

    //liste_age_personne = malloc(nombrePersonnes * sizeof(int)); // Valeurs aléatoires dans l'espace mémoire reservé.
    liste_age_personne = calloc(nombrePersonnes, sizeof(int));    // Des 0 dans l'espace mémoire réservé.

    if(liste_age_personne == NULL){ // Si problème lors de l'allocation dynamique, on arrête.
        exit(0);
    }

    for (int i = 0 ; i < nombrePersonnes ; i++){
        printf("Personne numero : %d et son age est %d \n",i, i*12);
        liste_age_personne[i] = i*12 ;
    }

    for (int i = 0 ; i < nombrePersonnes ; i++){
        printf(" [ %d ] ", liste_age_personne[i]);
    }
    printf("\n");

    /*--------------------------- realloc() ---------------------------*/

    nombrePersonnes = 5;
    liste_age_personne = realloc(liste_age_personne, nombrePersonnes * sizeof(int));
    for (int i = 0 ; i < nombrePersonnes ; i++){
        printf("Personne numero : %d et son age est %d \n",i, i*12);
        liste_age_personne[i] = i*12 ;
    }

    for (int i = 0 ; i < nombrePersonnes ; i++){
        printf(" [ %d ] ", liste_age_personne[i]);
    }
    printf("\n");

    free(liste_age_personne);

    return 0;
}

Fichiers​

Fonctions d’ouverture et la fermeture de fichiers :

  • fopen(<fichier>, <mode_ouverture>) : ouvrir un fichier
    • Les modes d’ouvertures sont :
      • r : lecture seule, fichier doit exister
      • w : écriture seule + écrase, crée si inexistant
      • a : écriture seule + ajout en fin de fichier, créé si inexistant
      • r+ : lecture/écriture, fichier doit exister
      • w+ : lecture/écriture+ écrase, crée si inexistant
      • a+ : ajout lecture/écriture, ajout en fin de fichier, crée si inexistant
  • fclose(<fichier>) : fermer un fichier ouvert
  • feof(<fichier>) : tester la fin d'un fichier

Fonctions de lecture :

  • fgetc(<fichier>) : lire un caractère
  • fgets(<chaine>, <taille_chaine>, <fichier>) : lire une ligne
  • fscanf(<fichier>, <format>, ...) : lire un texte formaté

Fonctions d’écriture :

  • fputc(<caractere>, <fichier>) : ecrit un caractère
  • fputs(<chaine>, <fichier>) : ecrit une chaine de caractères
  • fprintf(<fichier>, <format>, ...) : ecrit du texte formaté

Fonctions de positionnement de curseur :

  • ftell(<fichier>) : retourne position curseur dans fichier
  • fseek(<fichier>, <deplacement>, <origine>) : déplacement curseur
    • <origine> peut valoir :
      • SEEK_SET : début fichier
      • SEEK_CUR : position courante
      • SEEK_END : fin fichier
  • rewind(<fichier>) : réinitialise la position du curseur (début du fichier)

Fonctions diverses :

  • rename(<ancien_nom>, <nouveau_nom>) : renommer
  • remove(<nom_fichier>) : supprimer (attention : irréversible)

#include <stdio.h>

int nbLignesFichier(FILE *fichier)
{
    char buffer[1024];
    int nbLignes = 0;

    while(fgets(buffer, sizeof(buffer), fichier) != NULL)
    {
        nbLignes++;
    }
    rewind(fichier);
    //Pareil que rewind() : fseek(fichier, 0, SEEK_SET);
    return nbLignes;
}

int main(){

    /** LECTURE **/
    FILE *fic = fopen("texte.txt","r");
    char lettre;
    char ligne[256];
    char mot_A[50];
    char mot_B[50];
    char ignore[50];

    if(fic == NULL){
        printf("Fichier non accessible");
        exit(0);
    } // On n'a pas réussi à ouvrir le fichier

    printf("Nombre de lignes : %d \n",nbLignesFichier(fic));

    // Lecture caractère par caractère
    while((lettre = fgetc(fic)) != EOF)
    {
        printf("%c_", lettre);
    }
    printf("\n");

    // Retour du curseur au début du fichier
    rewind(fic);

    // Lecture ligne par ligne
    while(fgets(ligne, 256, fic) != NULL){
        printf("Ligne vaut : %s", ligne);
    }

    // Retour du curseur au début du fichier
    rewind(fic);

    // Lecture formatée
    fscanf(fic, "%s %s %s %s",mot_A ,ignore ,ignore, mot_B);
    printf("Mot 1 : %s \n", mot_A);
    printf("Mot 4 : %s \n", mot_B);

    fclose(fic);

    return 0;
}
