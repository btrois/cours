Modèle OSI et TCP/IP

Modèle OSI​

Expliquons le modèle OSI théoriquement mais aussi avec l’analogie de l’envoi d’une lettre par voie postale.

Voici les 7 couches du modèle OSI :

1. Couche Physique (Physical Layer)​

Rôle : Transmission des données brutes sous forme de signaux électriques, optiques ou radio par câble Ethernet, fibre optique ou Wi-Fi.

Analogie : Les routes, les voies, ou les réseaux de transport utilisés par les camions ou les avions pour acheminer la lettre. Cette couche s'occupe de la connexion physique entre deux points.

2. Couche Liaison de Données (Data Link Layer)​

Rôle : Garantir un transfert de données sans erreur entre deux nœuds connectés directement.

Analogie : L’organisation des lettres dans un sac postal spécifique à votre ville. Elle veille à ce que la lettre soit correctement "emballée" pour ce transport particulier et gère les erreurs locales, comme une lettre mal scellée.

3. Couche Réseau (Network Layer)​

Rôle : Déterminer le chemin que les données (ou la lettre) doivent prendre pour atteindre leur destination.

Analogie : Le centre de tri régional ou national qui détermine quelle route ou quel transport (avion, train, camion) utilisera votre lettre pour arriver à la ville de votre destinataire.

4. Couche Transport (Transport Layer)​

Rôle : Gérer le transport de bout en bout, s’assurer que toutes les données arrivent dans le bon ordre et sans erreur.

Analogie : Un responsable au bureau de poste qui vérifie que chaque lettre est bien suivie. Si une lettre est perdue ou endommagée, il s'assure qu'elle est renvoyée ou réexpédiée.

5. Couche Session (Session Layer)​

Rôle : Gérer les connexions (sessions) entre deux applications, s’assurer qu'elles restent ouvertes le temps nécessaire pour échanger les données.

Analogie : Etablir une communication entre l’émetteur et le destinataire. Par exemple, s’assurer que quelqu’un est là pour recevoir la lettre ou ouvrir la boîte aux lettres.

6. Couche Présentation (Presentation Layer)​

Rôle : Traduire les données pour qu’elles soient compréhensibles (compression, chiffrement, etc.).

Analogie : Traduire la lettre dans une langue que le destinataire peut lire ou utiliser une enveloppe spéciale pour préserver la confidentialité (chiffrement).

7. Couche Application (Application Layer)​

Rôle : Fournir les services directement utilisables par l’utilisateur final.

Analogie : Ecrire et lire la lettre elle-même. Vous (l’utilisateur) êtes à cette couche : c’est là que se trouve l’interaction directe.

Modèle TCP/IP​

Le modèle TCP/IP, contrairement au modèle OSI à 7 couches, est plus simple et fonctionnel. Il contient 4 couches, chacune couvrant un ou plusieurs aspects du modèle OSI.

Expliquons le modèle TCP/IP théoriquement mais aussi avec l’analogie de l’envoi d’une lettre par vie postale.

Voici les 4 couches du modèle TCP/IP :

1. Couche Accès au Réseau (Network Access Layer)​

Correspond aux couches Physique et Liaison de Données du modèle OSI.

Rôle : S’occupe de l’envoi des données sur le réseau local via des supports physiques (câbles, Wi-Fi) et garanti un transfert de données sans erreur entre deux nœuds connectés directement.

Analogie : Les routes et infrastructures physiques qui permettent aux camions ou véhicules d’acheminer les lettres. C’est aussi le processus d’emballage et de mise dans un sac postal au départ.

2. Couche Internet (Internet Layer)​

Correspond à la couche Réseau du modèle OSI.

Rôle : Responsable du routage des paquets entre les différents réseaux pour atteindre leur destination.

Analogie : Le centre de tri régional ou national qui détermine quelle route ou quel transport (avion, train, camion) utilisera votre lettre pour arriver à la ville de votre destinataire.

3. Couche Transport (Transport Layer)​

Correspond à la couche Transport du modèle OSI.

Rôle : Gérer le transport de bout en bout, s’assurer que toutes les données arrivent dans le bon ordre et sans erreur.

Analogie : Un responsable au bureau de poste qui vérifie que chaque lettre est bien suivie. Si une lettre est perdue ou endommagée, il s'assure qu'elle est renvoyée ou réexpédiée.

4. Couche Application (Application Layer)​

Correspond aux couches Session, Présentation et Application du modèle OSI.

Rôle : Fournir les services directement utilisables par l’utilisateur final.

Analogie : Ecrire et lire la lettre elle-même. Vous (l’utilisateur) êtes à cette couche : c’est là que se trouve l’interaction directe.

Récapitulatif - Théorie​

4ApplicationInterface utilisateur et services applicatifs.
3TransportFiabilité, ordre et intégrité des données de bout en bout.
2InternetRoutage des données entre réseaux différents.
1Accès réseauTransmission physique des signaux et transfert sans erreur entre deux machines locales.

Récapitulatif - Analogie​

4ApplicationLa lettre et son contenu (langue + format + synchro.).
3TransportLe suivi et la gestion des erreurs dans l’acheminement.
2InternetLa planification de l’itinéraire vers la ville cible.
1Accès réseauLes routes, le moyen de transport et le sac postal pour le transport local.

Compléments nécessaires​

Encapsulation​

Le principe d'encapsulation consiste à envelopper les données à chaque couche du modèle réseau avec des informations spécifiques (en-têtes) nécessaires à leur transmission, pour qu'elles soient correctement interprétées et acheminées jusqu'à leur destination.

Ce principe s’applique le plus souvent au modèle TCP/IP, en voici une représentation schématique :

Pour ce qui est des protocoles, le plus souvent on retrouvera les suivants :

  • Ethernet pour la couche Accès réseau.
  • IP pour la couche Internet.
  • TCP ou UDP pour la couche Transport.

En revanche, le protocole de la couche Application va très souvent varier selon les besoins : HTTP, SMTP, FTP, …

Lors de l’émission, on encapsule. Lors de la réception, on décapsule.

Unités de données échangées​

Les données échangées d’une couche à une autre ont parfois un nom.

  • Couche Application : Pas de nom spécifique, on parle de “données” au sens générique du terme.
    • entête application + données utiles
  • Couche Transport : On parle de “Segment”.
    • entête transport + entête application + données utiles
  • Couche Internet : On parle de “Paquet” :
    • entête internet + entête transport + entête application + données utiles
  • Couche Accès réseau : On parle “Trame” :
    • entête accès réseau + entête internet + entête transport + entête application + données utiles

Le protocole ARP opère à la couche 2 et à la couche 3, on peut alors le considérer comme un protocole de couche 2,5.

Équipements réseau​

  • Hub :
    • Couche 1 / Physique - du modèle OSI.
    • Le hub transmet ce qu’il reçoit sur un port à tous ses autres ports. Et ce peu importe le destinataire mentionné.
    • Le hub ne sait ni lire les adresses MAC, ni lire les adresses IP.
  • Switch :
    • Couche 2 / Liaison de données - du modèle OSI.
    • Le switch permet de connecter les équipements au sein d’un réseau.
    • Le switch sait lire les adresses MAC, pas les adresses IP.
  • Routeur :
    • Couche 3 / Réseau - du modèle OSI.
    • Le routeur permet d’interconnecter plusieurs réseaux.
    • Le routeur sait lire les adresses IP, pas les adresses MAC.
